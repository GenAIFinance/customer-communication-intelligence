# Power BI Local Reporting — Optional

> **Important:** Power BI is entirely optional and local-only.
> The main public demo is always the Streamlit dashboard.
> Never commit `.pbix` files or any data to this repo.

---

## What This Is

A local Power BI Desktop report for generating professional-looking
screenshots to include in the GitHub README and CV/portfolio.

---

## Prerequisites

- Power BI Desktop (free): https://powerbi.microsoft.com/desktop
- The processed CSV file: `data/processed/communications_processed.csv`
  (generated by running `python -m src.data.ingest`)

---

## How to Use

1. Run the data pipeline to generate the CSV:
   ```bash
   python -m src.data.ingest
   ```

2. Open Power BI Desktop

3. Open `Customer_Communication_Intelligence.pbix`
   (or create a new report and connect to the CSV)

4. Connect to data source:
   - **Get Data → Text/CSV**
   - Navigate to `data/processed/communications_processed.csv`
   - Click **Load**

5. Refresh the report data

6. Build or review report pages:
   - Overview (KPIs, engagement distribution)
   - Segment Analysis (breakdown by segment and channel)
   - Campaign Performance (open rates, complaint rates)

---

## Exporting Screenshots

1. Navigate to the page you want to capture
2. **File → Export → Export to PDF** (for multi-page)
   or use Windows Snipping Tool for individual page PNGs
3. Save screenshots to:
   ```
   assets/screenshots/powerbi/powerbi_overview.png
   ```

---

## What to Commit to GitHub

✅ Screenshots only — `assets/screenshots/powerbi/*.png`
❌ Never commit the `.pbix` file (excluded in `.gitignore`)
❌ Never commit any CSV or data files
